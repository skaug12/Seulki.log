# Output을 MD 파일로 정리

직전 작업의 출력 결과를 구조화된 Markdown 파일로 정리하여 저장합니다.

## 사용법

```
/save-output [저장경로 또는 파일명]
```

- 인자 없이 실행하면 `note/출력기록/` 폴더에 자동 저장
- 파일명만 지정하면 `note/출력기록/` 폴더에 해당 이름으로 저장
- 전체 경로를 지정하면 해당 경로에 저장

## 실행

$ARGUMENTS를 참고하여 저장 경로를 결정합니다.

1. 직전 작업에서 생성된 출력(output)을 식별합니다:
   - 스크립트 실행 결과
   - API 응답
   - 데이터 처리 결과
   - 분석/생성된 텍스트
   - 명령어 실행 결과
   - 기타 Claude가 생성한 콘텐츠

2. 출력 내용의 유형을 판단하고, 유형에 맞는 형식으로 정리합니다:

### 유형 A: 데이터/결과 출력

```markdown
---
title: '[작업명] 실행 결과'
date: 'YYYY-MM-DD HH:MM'
type: output-log
source: '[스크립트명 또는 작업 설명]'
tags: []
---

# [작업명] 실행 결과

> **실행일시**: YYYY-MM-DD HH:MM
> **소스**: [실행한 스크립트/명령어]
> **상태**: 성공 / 부분성공 / 실패

---

## 실행 요약

[무엇을 실행했고, 어떤 결과가 나왔는지 2-3문장으로 요약]

---

## 실행 결과

[출력 내용을 구조화하여 정리]
[테이블, 리스트, 코드블록 등 적절한 마크다운 형식 활용]

---

## 세부 로그

```
[원본 출력 중 중요한 부분]
```

---

## 비고

[특이사항, 에러, 후속 조치 필요 사항 등]
```

### 유형 B: 생성된 콘텐츠 출력

```markdown
---
title: '[콘텐츠 제목]'
date: 'YYYY-MM-DD'
type: output-content
source: '[생성 방법/프롬프트 요약]'
tags: []
---

# [콘텐츠 제목]

> **생성일**: YYYY-MM-DD
> **유형**: [리뷰/분석/요약/번역/기타]

---

[생성된 콘텐츠 본문]
```

3. 정리 규칙:
   - 원본 출력의 핵심 정보를 빠짐없이 보존
   - 데이터는 가능한 테이블 형식으로 정리
   - 긴 로그는 핵심 부분만 발췌하고, 필요시 전체 로그를 접기(details) 태그로 포함
   - 에러 메시지는 코드 블록으로 보존
   - 수치 데이터는 정확히 기록
   - tags에는 관련 기술/도구/주제를 넣음

4. `note/출력기록/` 폴더가 없으면 생성합니다.

5. 기본 파일명: `YYYY-MM-DD_[작업요약].md`

6. 저장 완료 후 파일 경로와 간단한 요약을 사용자에게 알려줍니다.
