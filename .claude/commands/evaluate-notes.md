# HFK 상품 노트 완성도 평가

지정된 카테고리(시즌)의 상품 노트들을 가이드라인 기준으로 평가합니다.

## 사용법

```
/evaluate-notes
```

## 실행

1. 사용자에게 평가할 카테고리를 질문합니다:
   - 현재 시즌: 26봄주중, 26봄주말
   - 또는 특정 카테고리 직접 입력

2. 해당 카테고리의 모든 노트 파일을 읽습니다:
   ```
   product/{카테고리}/*.md
   ```

3. 각 노트를 평가 기준에 따라 점수를 매깁니다.

4. 결과를 테이블 형식으로 출력합니다.

5. 주요 이슈를 아이폰 미리알림 "HFK" 리스트에 추가합니다:
   - 긴급 수정 필요 항목 (오늘 18:00)
   - 70점 미만 노트의 이슈 (내일 09:00)

---

## 평가 기준 (100점 만점)

### 필수 섹션 (각 10점, 총 70점)

| 섹션 | 기준 | 배점 |
|------|------|------|
| 헤드라인 | 2줄, 임팩트 있는 문구 | 10점 |
| 소개 | 3문단 (공감-솔루션-초대) | 10점 |
| 01 팀 배경 | 3개 이상 포인트 | 10점 |
| 02 팀 운영 방향 | 3개 이상 포인트 | 10점 |
| 03 팀 성장 포인트 | 3개 이상 포인트 | 10점 |
| 04 팀 멤버들 | 3-4개 포인트 | 10점 |
| 진행방식 | 3개 포인트 | 10점 |

### 추가 섹션 (각 15점, 총 30점)

| 섹션 | 기준 | 배점 |
|------|------|------|
| 파트너 소개 | 이름, 소개, HFK 기획 이유 완성 | 15점 |
| 세션 주제 | 6회차 모두 주제+설명 작성 | 15점 |

---

## 감점 요소

| 이슈 | 감점 |
|------|------|
| 헤드라인 1줄만 있음 | -5점 |
| 헤드라인 불완전 (문장 미완성) | -10점 |
| 섹션 포인트 2개만 있음 | -5점 |
| 섹션 포인트 1개만 있음 | -8점 |
| 불릿 포인트 형식 깨짐 | -5점 |
| 세션 주제만 있고 설명 없음 | -10점 |
| 세션 주제/설명 모두 비어있음 | -15점 |
| 파트너 소개 간략함 | -5점 |
| 다른 팀 내용 복사됨 | -50점 |
| 팀명 오기재 | -10점 |
| 파트너 이름 오류 | -10점 |

## 출력 형식

### 요약 테이블

```markdown
## {카테고리} 완성도 평가 ({N}개)

| 팀명 | 점수 | 주요 이슈 |
|------|------|----------|
| 팀A | 100 | 완성 |
| 팀B | 70 | 세션 주제 미완성 |
| 팀C | 40 | 다른 팀 내용 복사됨 |
```

### 긴급 수정 필요 항목

치명적 오류가 있는 노트를 별도로 표시:
- 다른 팀 내용이 복사된 경우
- 팀명이 잘못 기재된 경우
- 파트너 이름이 다른 사람으로 되어있는 경우

## 미리알림 추가

평가 완료 후, 주요 이슈를 아이폰 미리알림에 자동으로 추가합니다.

### 미리알림 생성 규칙

1. **긴급 수정 필요** 항목은 반드시 미리알림에 추가:
   - 다른 팀 내용 복사됨
   - 팀명 오기재
   - 파트너 이름 오류

2. **70점 미만** 노트의 주요 이슈도 미리알림에 추가

3. 미리알림 형식:
   ```
   [팀명] 이슈 내용
   ```
   예시:
   - `[마케팅디깅] 경영브릿지 내용 복사됨 - 전면 수정 필요`
   - `[경영에센셜] 소개문 팀명 오류 수정`
   - `[리더의서재] 세션 주제 작성 필요`

### 미리알림 추가

MCP Apple Reminders 도구를 사용합니다:
- 여러 이슈: `bulk_create` 도구 사용
- 리스트: "HFK"
- 형식: `[팀명] 이슈 내용`

### 마감일 설정

- 기본값: 내일 오전 9시 (ISO 8601)
- 긴급 수정 필요 항목: 오늘 오후 6시
