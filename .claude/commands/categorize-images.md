# 이미지 분류 스킬

시즌(카테고리)별 팀에 어울리는 이미지를 분류합니다.

## 사용법

1. `/Users/sklee/hfk-imweb/images/source/` 폴더에 이미지 파일을 넣습니다
2. `/categorize-images` 스킬을 실행합니다
3. 시즌을 선택하고 팀 정보를 확인합니다
4. 확인 후 각 팀 폴더에 4개씩 이미지가 분류됩니다

## 실행 단계

### 0단계: 시즌 선택

사용자에게 분류할 시즌(카테고리)을 선택받습니다.

**사용 가능한 시즌 목록**:
- 현재: 26봄주중, 26봄주말
- 지난 시즌: 25겨울주중, 25겨울주말, 25여름주중, 25여름주말, 25봄주중, 25봄주말, 24겨울, 24가을

AskUserQuestion 도구를 사용하여 시즌을 선택받습니다:
- header: "시즌 선택"
- question: "이미지를 분류할 시즌을 선택하세요."
- options: 현재 시즌 우선 표시

### 1단계: 팀 정보 수집

선택된 시즌의 상품 노트를 읽고, 각 팀의 주제와 설명을 파악합니다:

- `/Users/sklee/hfk-imweb/product/{선택된시즌}/` 폴더의 모든 .md 파일

각 팀의 핵심 키워드와 테마를 추출합니다 (헤드라인, 소개, 팀 배경 섹션 참고).

### 1.5단계: 정보 확인

추출한 팀 정보를 사용자에게 보여주고 확인받습니다:
- 시즌명
- 팀 목록 (팀명, 핵심 키워드)
- source 폴더의 이미지 개수

AskUserQuestion으로 진행 여부를 확인합니다:
- header: "분류 확인"
- question: "위 정보로 이미지 분류를 진행할까요?"
- options: "진행", "취소"

### 2단계: 이미지 분석

`/Users/sklee/hfk-imweb/images/source/` 폴더의 모든 이미지를 분석합니다:

- 이미지의 주제, 분위기, 색감, 포함된 요소 파악
- 비즈니스/전문성/창의성/감성 등의 특성 분류

### 3단계: 매칭 및 분류

각 팀에 가장 어울리는 이미지 4개를 선정합니다:

- 팀 테마와 이미지 특성 매칭
- 다양성 고려 (비슷한 이미지 중복 방지)
- 각 팀 폴더 생성: `/Users/sklee/hfk-imweb/images/{시즌}/{팀명}/`
- 선정된 이미지를 해당 폴더로 복사

### 4단계: 결과 리포트

분류 결과를 보고합니다:
- 시즌 및 팀 수
- 각 팀별 선정된 이미지 목록
- 선정 이유 간략 설명
- 미분류된 이미지 목록 (있을 경우)

## 참고

- 이미지는 원본 보존 (복사, 이동 아님)
- 한 이미지가 여러 팀에 적합할 경우 가장 적합한 팀 하나에만 배정
- 팀당 정확히 4개 이미지 필요 (부족시 가장 유사한 이미지로 보충)
