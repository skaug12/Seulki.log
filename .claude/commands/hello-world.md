# Hello World - 하루 시작 에이전트

하루를 시작할 때 실행. 코드 동기화 → 스킬 현황 → 캘린더 → 미리알림 → 어제 이어서 할 일까지, 아무것도 놓치지 않게 브리핑합니다.

## 사용법

```
/hello-world
```

## 실행 워크플로우

### Step 1: git pull (최신 코드 동기화)

```bash
cd $HOME/Seulki.log && git pull origin main
```

- 충돌이 있으면 사용자에게 알리고 중단
- 성공 시 변경된 파일 목록 간단히 표시

### Step 2: 스킬 & 에이전트 현황 요약

`$HOME/Seulki.log/.claude/SKILL_GUIDE.md`를 읽고 아래 형식으로 요약 출력:

```
📋 스킬 현황
━━━━━━━━━━━━━━━━━━━━━━
에이전트 5개 | 개별 스킬 16개 | MCP 서버 4개

에이전트: /run-season-ops, /run-product-ops, /run-content-ops, /run-image-ops, /run-daily-note
MCP: Calendar ✓ | Slack ✓ | Reminders ✓ | Imweb ✓
```

- 스킬 수는 SKILL_GUIDE.md에서 실제로 세어서 표시
- MCP 서버 연결 상태는 목록만 표시 (실제 health check 불필요)

### Step 3: 오늘 & 이번 주 캘린더 확인

MCP Google Calendar로 일정을 조회합니다.

1. **오늘 일정**: 오늘 날짜의 모든 이벤트
2. **이번 주 남은 일정**: 오늘 이후 ~ 이번 주 일요일까지의 이벤트
3. 아래 형식으로 출력:

```
📅 오늘 (3/15 토)
  - 없음

📅 이번 주 남은 일정
  - 3/16 (일) 14:00 영화로운일 세션
  - 3/16 (일) 14:00 넘버앤센스 세션
```

- 일정이 없으면 "없음"으로 표시
- 세션 날짜가 가까우면 (3일 이내) 강조 표시

### Step 4: 미리알림 브리핑

MCP Apple Reminders로 "HFK" 리스트를 조회합니다.

1. **미완료 항목** 전체 조회
2. 마감일 기준으로 정렬하여 출력:

```
⏰ 미리알림 (미완료 N개)
━━━━━━━━━━━━━━━━━━━━━━
🔴 지난 것 (overdue)
  - [3/13] 핸드아웃 업데이트
  - [3/14] 인스타 포스팅

🟡 오늘/내일
  - [3/15] 시즌레터 검토

🟢 이번 주
  - [3/18] 상품 동기화
```

- 제외 필터: "AI부사수", "필사", "에세이" 포함 항목은 제외
- overdue 항목이 있으면 강조

### Step 5: 어제 데일리 노트 확인

1. `$HOME/Seulki.log/daily/` 에서 가장 최근 데일리 노트를 찾습니다
2. 해당 파일의 "오늘 완료한 작업"과 "메모" 섹션을 읽어서 요약:

```
📝 최근 작업 (3/14)
━━━━━━━━━━━━━━━━━━━━━━
  - 상품 노트 3개 동기화 완료
  - 스크린샷 OCR 스킬 버그 수정
  메모: "내일 시즌레터 마무리할 것"
```

- 파일이 없으면 "최근 데일리 노트 없음"으로 표시

### Step 6: 오늘의 루틴 안내

오늘 요일에 따라 실행할 수 있는 에이전트/스킬을 안내합니다.

| 요일 | 안내 |
|------|------|
| 월 | "이번 주 시작! 미리알림 점검 완료. 필요하면 `/run-product-ops`" |
| 화 | "내일 수요일 루틴 준비. 슬랙 확인해둘 것" |
| 수 | "**수요일 루틴 실행 가능**: `/run-wednesday`" |
| 목 | "핸드아웃/공지 마감 확인. `/schedule-notices` 필요하면 실행" |
| 금 | "주간 마무리. `/run-daily-note`로 정리 추천" |
| 토-일 | "주말. 세션이 있다면 캘린더 확인 완료" |

### Step 7: 브리핑 완료

모든 정보를 종합하여 한 줄 요약:

```
━━━━━━━━━━━━━━━━━━━━━━
✅ 브리핑 완료
  일정 N개 | 미리알림 N개 (overdue N개) | 오늘: {요일별 한마디}
━━━━━━━━━━━━━━━━━━━━━━
```

## 워크플로우 다이어그램

```
[Step 1: git pull] ← 코드 동기화
      ↓
[Step 2: 스킬 현황] ← SKILL_GUIDE.md
      ↓
[Step 3: 캘린더] ← Google Calendar MCP
      ↓
[Step 4: 미리알림] ← Apple Reminders MCP
      ↓
[Step 5: 최근 데일리] ← daily/*.md
      ↓
[Step 6: 오늘 루틴 안내] ← 요일 기반
      ↓
[Step 7: 브리핑 완료 요약]
```

## 데이터 소스

| 정보 | 소스 | 인증 |
|------|------|------|
| 코드 | git (Seulki.log) | SSH key |
| 스킬 현황 | SKILL_GUIDE.md | 파일 시스템 |
| 캘린더 | Google Calendar | MCP (OAuth) |
| 미리알림 | Apple Reminders | MCP (macOS) |
| 작업 이력 | daily/*.md | 파일 시스템 |

## 참고

- 모든 Step은 자동 실행 (사용자 확인 없이 순차 진행)
- 오류가 나도 다음 Step으로 계속 진행 (브리핑이 끊기지 않도록)
- git pull 충돌 시에만 중단하고 사용자에게 알림
