# HFK 시즌 운영 에이전트

시즌 시작/운영에 필요한 일련의 작업을 자동으로 수행하는 에이전트입니다.
개별 스킬을 순차적으로 실행하며, 사용자 승인이 필요한 지점에서 확인을 요청합니다.

## 사용법

```
/run-season-ops
```

## 실행 워크플로우

### Step 1: 일정 업데이트 (update-schedule)

1. 사용자에게 대상 시즌 카테고리를 선택하도록 질문합니다 (26봄주중, 26봄주말 등)
2. MCP Google Calendar `list-events`로 해당 시즌 기간의 캘린더 일정을 조회합니다
3. 이벤트를 팀명별로 매칭하여 각 상품 노트의 세션 날짜를 업데이트합니다
4. 업데이트된 파일 목록과 변경 내용을 사용자에게 보여줍니다
5. `status`를 `draft`로 변경합니다

**사용자 확인**: 업데이트된 일정이 맞는지 확인

### Step 2: 아임웹 동기화 (sync-products)

1. Step 1에서 `status: draft`로 변경된 파일 목록을 보여줍니다
2. 각 파일의 Markdown을 파싱하여 HTML 생성
3. MCP Imweb `imweb_update_product` / `imweb_create_product`로 동기화
4. 성공 시 `status: synced`로 변경

**사용자 확인**: 동기화 진행 여부

### Step 3: 시즌레터 생성 (generate-letter) [선택]

사용자에게 시즌레터를 생성할지 질문합니다:
- "예" → 시즌, 레터번호, 제목을 입력받아 시즌레터 생성
  - MCP Google Calendar `list-events`로 팀 세션 안내 일정 자동 조회
  - 섹션 구성에 대해 사용자와 대화
  - HTML 결과물 생성
- "아니오" → Step 4로 이동

### Step 4: Slack 공지 예약 (schedule-notices) [선택]

사용자에게 Slack 공지를 예약할지 질문합니다:
- "예" → 시작일, 종료일, 예약발송일시를 입력받아 실행
  - MCP Google Calendar로 일정 조회
  - MCP Slack `slack_post_message`로 테스트 DM 발송
  - 사용자 승인 후 실제 채널에 발송
- "아니오" → Step 5로 이동

### Step 5: 전환 분석 (analyze-conversion) [선택]

아임웹 회원 중 HFK 미등록자/봄시즌 미등록자를 분석하여 전환 대상 리스트를 생성합니다.

사용자에게 전환 분석을 실행할지 질문합니다:
- "예" → 아래 절차 실행
- "아니오" → 종료

#### 사전 준비
- 아임웹 관리자 > 회원관리 > CSV 내보내기 필요 (API는 100명 제한)
- 사용자에게 `~/Downloads/`에 CSV 파일이 있는지 확인

#### 실행 절차

1. **데이터 수집**
   - 아임웹 회원 CSV 파싱 (이메일, 이름, 가입일, 최근로그인, 로그인횟수, 구매횟수, 구매금액)
   - HFK 통합멤버십 멤버 리스트 조회 (Google Sheets ID: `1BydHQ0DQagXdzhVZ1QD6JPO8dQ0eYZPQ68nipw0B8WE`, `멤버 리스트` 시트, L열=이메일, D열=시즌)

2. **교차 비교** (이메일 lowercase 매칭)
   - HFK 한번도 등록 X → 미등록자 리스트
   - HFK 이력 O, 현재 시즌 X → 봄시즌 미등록 리스트

3. **우선순위 점수 산정** (100점 만점)

   | 요소 | 배점 | 기준 |
   |------|------|------|
   | 최근 로그인 | 최대 40점 | 7일내 40 / 30일내 30 / 90일내 20 / 180일내 10 / 1년내 5 |
   | 로그인 횟수 | 최대 20점 | 50회+ 20 / 20회+ 15 / 10회+ 10 / 5회+ 5 |
   | 구매 횟수 | 최대 15점 | 3회+ 15 / 1회+ 10 |
   | 구매 금액 | 최대 10점 | 100만+ 10 / 50만+ 5 |
   | 재등록 대상 | +15점 | HFK 이전 등록 이력 있을 때 가산 |

4. **세그먼트 분류**

   미등록자:
   - A. 구매경험+최근활동 (최우선) → 개인 DM + 시즌 안내
   - B. 구매경험+비활동 → 구매 감사 메일 + 시즌 혜택
   - C. 최근활동+반복방문 (높음) → 얼리버드 할인 + 팀 소개
   - D. 최근활동 (중간) → 시즌 하이라이트 콘텐츠
   - E. 1년내 활동 (낮음) → 시즌 오픈 알림 이메일
   - F. 비활동 (관망) → 재활성화 콘텐츠 후 판단

   재등록대상:
   - A. 최근활동 재등록대상 (30일내 로그인)
   - B. 반년내 활동 재등록대상 (180일내)
   - C. 비활동 재등록대상 (180일+)

5. **결과 출력** — Google Sheets에 4개 시트 생성
   - `전환분석 요약`: 전체 현황 + 세그먼트별 인원 + 전환 전략
   - `미등록자 리스트`: 우선순위순 전체 리스트
   - `봄시즌 미등록`: 참여시즌수/마지막시즌 포함
   - `이메일 발송 리스트`: 미등록 20점+, 재등록 30점+ 대상 + 추천 메시지 유형

**사용자 확인**: 결과 시트 URL 지정 (서비스 계정 `hfkmember@hfkmember.iam.gserviceaccount.com`에 편집 권한 필요)

#### 참고
- 분석 상세: `docs/전환분석_2026봄.md`
- 아임웹 API 인증: `$HOME/imweb-order-sync/.env`
- 아임웹 API 회원 조회는 100명 제한 → CSV 필수

## 워크플로우 다이어그램

```
[카테고리 선택]
      ↓
[Step 1: 일정 업데이트] ← Google Calendar MCP
      ↓ (사용자 확인)
[Step 2: 아임웹 동기화] ← Imweb MCP
      ↓ (사용자 확인)
[Step 3: 시즌레터?] ← Google Calendar MCP (선택)
      ↓
[Step 4: Slack 공지?] ← Slack MCP (선택)
      ↓
[Step 5: 전환 분석?] ← 아임웹 CSV + Google Sheets (선택)
      ↓
[완료 요약]
```

## 완료 요약

- 업데이트된 상품 노트 수
- 동기화된 상품 수
- 생성된 시즌레터 (있는 경우)
- 예약된 Slack 공지 수 (있는 경우)
- 전환 분석 결과: 미등록자/봄시즌미등록/이메일발송 대상 수 (있는 경우)
