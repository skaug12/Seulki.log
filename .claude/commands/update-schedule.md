# HFK 일정 업데이트

구글 캘린더에서 HFK 일정을 가져와 상품 정보를 업데이트합니다.

## 사용법

```
/update-schedule
```

## 실행

1. 사용자에게 업데이트할 카테고리(시즌)를 선택하도록 질문합니다:
   - 현재 시즌: 26봄주중, 26봄주말

2. MCP Google Calendar `list-events` 도구로 해당 시즌 일정을 조회합니다.

3. 조회된 이벤트를 팀명/날짜로 매칭하여 각 노트 파일의 세션 날짜를 직접 업데이트합니다.

4. 업데이트된 파일 목록을 보여줍니다.

4. 변경된 노트의 `status`가 자동으로 `draft`로 변경됩니다.

5. `/sync-products`로 아임웹에 동기화할 수 있음을 안내합니다.

---

## 캘린더 조회

MCP Google Calendar 도구를 사용합니다:
- `list-events`: 시즌 기간의 전체 이벤트 조회
- `search-events`: 특정 팀명으로 이벤트 검색

---

## 일정 매칭 규칙

캘린더 이벤트 제목에서 팀명을 추출하여 자동 매칭합니다:

| 캘린더 이벤트 형식 | 매칭되는 노트 |
|-------------------|------------|
| `[팀] 26봄 영화로운일` | 영화로운일.md |
| `[팀] 26봄 AI핸즈온` | AI핸즈온.md |
| `[팀] 강점차별화` | 강점차별화.md |

**매칭 우선순위:**
1. 직접 매칭: 노트 파일명 == 캘린더 팀명
2. 26봄 prefix 매칭: `26봄 {노트 파일명}` == 캘린더 팀명
3. 부분 매칭: 노트 파일명이 캘린더 팀명에 포함

**참고:** 파일명의 숫자 접두사(예: "1 써보는경험")는 자동으로 제거되어 매칭됩니다.

## 장소 자동 분류

일정 제목/장소에 키워드가 있으면 자동 분류됩니다:

| 키워드 | 장소 |
|--------|------|
| 오아시스, 덕수궁 | 오아시스 덕수궁 |
| 소정동 | 소정동 |

## 업데이트 내용

1. **schedule 필드**: 팀의 요일/시간 정보 업데이트
2. **세션별 날짜**: 각 회차의 날짜 자동 업데이트

### 예시

**업데이트 전:**
```markdown
---
schedule: 화요일 19:30-22:00 (3개월)
---

### 1회차
- **주제**: 세션 주제
```

**업데이트 후:**
```markdown
---
schedule: 일요일 10:30-13:00 (3개월)
---

### 1회차
- **날짜**: 3월 15일 (일)
- **주제**: 세션 주제
```

## 변경 범위 제한 (중요)

이 스킬은 **날짜 관련 필드만 업데이트**합니다. 아래 외의 내용은 절대 수정하지 않습니다:

### 변경하는 필드 (허용)
- frontmatter `schedule`: 요일/시간 정보
- 각 회차의 `- **날짜**:` 줄: 날짜 추가 또는 업데이트
- frontmatter `status`: `draft`로 변경

### 변경하지 않는 필드 (금지)
- 각 회차의 `- **주제**:` 내용
- 각 회차의 `- **설명**:` 내용
- 각 회차의 `- **활동**:` 및 하위 항목 내용
- 헤드라인, 소개, 팀 배경, 운영 방향, 성장 포인트, 팀 멤버들, 파트너 소개 등 모든 본문 섹션
- frontmatter의 `schedule`, `status` 외 모든 필드 (`images`, `price`, `name` 등)

### 편집 방식
- 파일 전체를 다시 쓰지 않고, **해당 줄만 찾아서 교체(Edit)**합니다
- 날짜 줄이 없는 회차에는 `- **주제**:` 줄 바로 위에 `- **날짜**:` 줄을 삽입합니다
- 기존 날짜가 있으면 해당 줄의 값만 교체합니다

## 주의사항

- 지난 일정(오늘 이전)은 조회에서 제외됩니다
- 캘린더 API 키가 유효해야 합니다
- 노트 업데이트 후 자동으로 아임웹에 동기화되지 않습니다 (별도로 `/sync-products` 실행 필요)
