# HFK 이미지 관리 에이전트

source 폴더의 이미지를 용도에 맞게 자동 분류합니다.
상품용 이미지와 이벤트 사진을 구분하여 적절한 분류 워크플로우를 실행합니다.

## 사용법

```
/run-image-ops
```

## 실행 워크플로우

### Step 1: 분류 대상 확인

다음 폴더에 분류 대기 중인 이미지가 있는지 확인합니다:
- 상품용: `$HFK_IMAGES/source/`
- HFK 사진: `$HFK_PHOTOS/`

각 폴더의 이미지 수를 보여줍니다.

### Step 2: 용도 선택

사용자에게 분류할 이미지 종류를 질문합니다:
- **상품용 이미지** → categorize-images-imweb 워크플로우
- **이벤트 사진** → categorize-images-hfk 워크플로우
- **둘 다** → 순차적으로 실행

### Step 3a: 상품용 이미지 분류 (categorize-images-imweb)

1. 시즌 선택 (26봄주중, 26봄주말 등)
2. 분류할 팀 선택 (다중 선택)
3. source 폴더 이미지 자동 리사이즈 (가로 1920px 이하)
4. Claude 비전으로 이미지 분석
5. 팀 테마와 매칭하여 팀당 4개 이미지 선정 및 복사

### Step 3b: 이벤트 사진 분류 (categorize-images-hfk)

1. `$HFK_PHOTOS/` 폴더의 사진 확인
2. 이미지 리사이즈 (가로 1920px 이하)
3. EXIF 데이터에서 촬영 날짜/시간 추출
4. MCP Google Calendar `list-events`로 HFK 일정 조회
5. 촬영 시간과 캘린더 일정 매칭
6. 매칭 결과 확인 후 이벤트별 폴더로 이동

**주의**: 사진은 이동(move)되므로 사용자에게 사전 확인

## 완료 요약

- 분류된 이미지 수
- 분류 결과 (팀/이벤트별 내역)
- 매칭 안 된 이미지 수 (_unmatched)
