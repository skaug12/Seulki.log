# HFK 녹음 → 콘텐츠 초안 생성

클로버노트에서 내보낸 녹취 텍스트 파일을 가져와 리뷰 초안을 생성하는 워크플로우입니다.

## 사용법

```
/draft-from-recording          # ~/Downloads/에서 녹취 파일 자동 탐색
/draft-from-recording [파일경로]  # 특정 파일 지정
```

## 실행

### Step 1: 다운로드 폴더에서 녹취 파일 가져오기

1. $ARGUMENTS가 있으면 해당 파일을 바로 Step 2로 넘깁니다.
2. 인자가 없으면 `~/Downloads/` 폴더에서 클로버노트 텍스트 파일(`.txt`)을 검색합니다:
   ```bash
   ls -t ~/Downloads/*.txt | head -5
   ```
3. 찾은 파일 목록을 사용자에게 보여주고 어떤 파일인지 확인합니다.
4. 확인되면 `recordings/` 폴더로 이동합니다:
   ```bash
   mv ~/Downloads/{선택된파일}.txt ~/Seulki.log/recordings/
   ```
5. 다운로드 폴더에 `.txt` 파일이 없으면 사용자에게 파일 경로를 직접 질문합니다.

### Step 2: 리뷰 유형 선택

사용자에게 질문합니다:

> 어떤 리뷰를 작성할까요?
> 1. **이벤트 리뷰** — 북토크, AAR, PEST 등 이벤트 후기
> 2. **팀 리뷰** — 팀 활동/세션 후기

### Step 3: 녹취 텍스트 전처리

1. 파일 내용을 읽습니다.
2. 클로버노트 특유의 형식을 정리합니다:
   - 화자 라벨 (화자1, 화자2 등) → 문맥에서 실제 이름 추론
   - 타임스탬프 제거
   - 중복/반복 발화 정리

### Step 4: 초안 생성

선택한 유형에 따라 해당 스킬을 실행합니다:

- **이벤트 리뷰** → `/generate-event-review` 스킬의 원칙에 따라 생성
- **팀 리뷰** → `/generate-team-review` 스킬의 프레임워크에 따라 생성

결과를 `$HFK_VAULT/note/리뷰/` 폴더에 저장합니다.
- 파일명: `{유형}_{날짜 또는 주제}.md`

**사용자 확인**: 생성된 초안 내용 검토

### Step 5: HTML 변환 (선택)

사용자에게 질문합니다:

> 아임웹 HTML로 변환할까요? (예/아니오)

- "예" → `/convert-to-imweb` 실행
- "아니오" → 종료

## 폴더 구조

```
recordings/       ← 클로버노트 텍스트 파일 저장 위치
  ├── 북토크_240315.txt
  ├── AAR_밋업_240320.txt
  ├── 팀_독서토론_240318.txt
  └── ...
```

## 참고

- 클로버노트 앱 → 녹음 선택 → 내보내기 → 텍스트(.txt)로 저장
- 파일명에 이벤트 유형을 포함하면 자동 추론이 더 정확합니다
