# HFK 아이디어 확장

아이디어 노트를 바탕으로 완성도 높은 상품 초안(draft)을 작성합니다.

## 인자

$ARGUMENTS: 아이디어 파일명 (예: 글쓰기모임)

## 프로세스

1. `product/아이디어/{파일명}.md` 읽기
2. HFK 톤앤매너에 맞춰 초안 작성
3. `product/아이디어/{파일명}_draft.md`로 새 파일 생성 (원본 유지)

## 작성 가이드라인

### 타겟 독자
- 서울의 30-40대 직장인 (대리~부장급)
- 실무 리더, 중간 관리자
- **핵심 니즈**: 직장에서 인정받고, 더 성장하고 싶은 욕구

### 톤앤매너
- 쉽고 구체적으로 (전문용어 지양)
- "이거 하면 직장에서 더 잘할 수 있겠다" 느낌
- 혼자가 아닌 '함께' 성장하는 가치 강조
- 실무에 바로 적용할 수 있는 실용성

### 섹션별 작성 요령

#### 헤드라인 (2줄)
- 시적이고 임팩트 있게
- 직장인의 고민을 건드리는 한마디
- 예: "팀을 넘어, 조직을 잇는 / 전략 리더가 필요합니다"

#### 소개 (3문단)
1. **공감**: 직장에서 겪는 구체적 고민 묘사
2. **솔루션**: 이 팀에서 어떻게 해결하는지
3. **초대**: 성장하고 싶은 사람에게 권유

#### 01 팀 배경 (3개)
- 왜 이런 팀이 필요한가?
- 직장인이 겪는 한계점 언급
- 구체적 상황 예시 포함

#### 02 팀 운영 방향 (3개)
- 매 세션 어떻게 진행되는지
- 파트너 역할, 멤버 간 상호작용
- 어떤 자료/도구를 활용하는지

#### 03 팀 성장 포인트 (3개)
- 참여하면 구체적으로 무엇을 얻는가?
- 직장에서 어떻게 인정받게 되는가?
- 실무에 바로 쓸 수 있는 것

#### 04 팀 멤버들 (3-4개)
- 어떤 상황의 사람이 오면 좋은가?
- "~한 분" 형식
- 구체적 직무/고민 언급

#### 파트너 소개
- 파트너 정보가 없으면 placeholder로 남김
- "파트너 정보 추가 필요" 표시

#### 3개월 세션 주제
- 6회차 구성
- 각 회차별 주제와 설명 작성
- 점진적으로 심화되는 커리큘럼
- 활동(토론/실습 등) 구체적으로

#### 진행방식 (3개)
- 기본 포맷: 15명 / 4L 회고 / 리캡→발표→토론→회고

### 완성도 체크리스트
- [ ] 헤드라인이 직장인의 마음을 건드리는가?
- [ ] 소개 3문단이 자연스럽게 이어지는가?
- [ ] 각 섹션이 3개 이상의 포인트를 갖추었는가?
- [ ] 세션 6회차가 논리적 흐름을 갖추었는가?
- [ ] "이거 들으면 직장에서 더 잘할 수 있겠다" 느낌이 드는가?
- [ ] 전문용어 없이 쉽게 읽히는가?

## 출력 형식

아이디어 파일을 읽고, 위 가이드라인에 맞춰 완성도 높은 초안을 작성한 뒤,
**`{파일명}_draft.md`로 새 파일을 생성**합니다. (원본 아이디어 노트는 그대로 유지)

Frontmatter:
```yaml
---
name: "팀명"
category: ""  # 사용자가 나중에 선택
schedule: ""  # 사용자가 나중에 결정
price: 590000
status: draft
product_no: ""
images: []
---
```

- product/26봄주중/*.md
- product/26봄주말/*.md
