# 대화 히스토리를 MD 파일로 정리

현재 대화 세션의 내용을 구조화된 Markdown 파일로 정리하여 저장합니다.

## 사용법

```
/save-conversation [저장경로 또는 파일명]
```

- 인자 없이 실행하면 `note/대화기록/` 폴더에 자동 저장
- 파일명만 지정하면 `note/대화기록/` 폴더에 해당 이름으로 저장
- 전체 경로를 지정하면 해당 경로에 저장

## 실행

$ARGUMENTS를 참고하여 저장 경로를 결정합니다.

1. 현재까지의 대화 내용을 전체적으로 검토합니다.

2. 아래 형식에 맞춰 대화 내용을 정리합니다:

```markdown
---
title: '[대화 주제 요약]'
date: 'YYYY-MM-DD'
type: conversation-log
tags: []
---

# [대화 주제 요약]

> **날짜**: YYYY-MM-DD
> **주요 주제**: 핵심 키워드 나열

---

## 요약

[전체 대화의 핵심 내용을 3-5문장으로 요약]

---

## 대화 흐름

### 1. [첫 번째 주제/요청]

**요청**: [사용자가 요청한 내용 요약]

**결과**: [수행된 작업과 결과 요약]

**주요 내용**:
- [핵심 포인트 1]
- [핵심 포인트 2]

### 2. [두 번째 주제/요청]

...

---

## 수행된 작업 목록

- [ ] 또는 - [x] 형태로 작업 목록 정리

---

## 참고사항

[대화 중 나온 중요한 참고사항, 링크, 파일 경로 등]
```

3. 정리 규칙:
   - 시간순으로 대화 흐름을 정리
   - 각 주제/요청별로 섹션을 나눔
   - 코드 스니펫은 코드 블록으로 보존
   - 파일 경로, 명령어 등 기술적 내용은 정확히 기록
   - 불필요한 반복이나 시행착오는 결과 중심으로 축약
   - 에러/해결 과정이 있으면 별도로 기록
   - tags에는 대화에서 다룬 기술/도구/주제를 넣음

4. `note/대화기록/` 폴더가 없으면 생성합니다.

5. 기본 파일명: `YYYY-MM-DD_[주제요약].md`

6. 저장 완료 후 파일 경로와 요약을 사용자에게 알려줍니다.
