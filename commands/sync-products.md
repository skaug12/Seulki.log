# HFK 상품 동기화

Obsidian에서 수정한 상품 노트를 아임웹에 동기화합니다.

## 사용법

```
/sync-products
```

## 실행

1. 사용자에게 동기화할 카테고리(시즌)를 선택하도록 질문합니다:
   - 현재 시즌: 26봄주중, 26봄주말, 추가등록, 이벤트, 시즌오프
   - 지난 시즌도 선택 가능

2. 선택한 카테고리의 노트들 중 `status: draft` 상태인 파일 목록을 보여줍니다.

3. 동기화 진행 여부를 확인합니다.

4. **신규 상품 생성 시 이미지 확인**: `product_no`가 없고 `images: []`인 파일은 기본 이미지 4개를 frontmatter에 추가합니다.

5. 확인 후 동기화를 실행합니다. 각 파일에 대해:
   - Markdown을 파싱하여 구조화된 데이터 추출
   - HTML 코드 생성 (content, simple_content)
   - MCP Imweb 도구로 상품 생성 또는 업데이트:
     - `product_no` 없음 → `imweb_create_product` 도구로 신규 생성
     - `product_no` 있음 → `imweb_update_product` 도구로 업데이트
   - 성공 시 frontmatter의 `status`를 `synced`로 변경

6. 실행 결과를 사용자에게 알려줍니다.

### 노트 구조 요구사항

동기화되려면 노트가 다음 구조를 따라야 합니다:

name: '팀명'

### ODC 멤버십 노트 구조

ODC 멤버십 상품은 별도의 노트 구조를 따릅니다:

```markdown
---
category: ODC멤버십
name: ODC 관계
price: 480000
prod_status: sale
product_no: '236'
status: draft  # draft여야 동기화됨
headline: 팀 없이, 연결만으로
subheadline: 이벤트와 클럽으로 HFK 커뮤니티에 머물고 싶은 분
team_count: 0개 시즌
simple_includes: 4개 시즌 이벤트 & 클럽 + 어드벤처
simple_comparison: 팀 0개 vs 4개 | 어드벤처 4회 vs 없음 | 480,000원 vs 156만원
simple_addon: 희망 시 팀 추가 등록 가능 (27만원 할인)
images:
  - https://cdn.imweb.me/...
gallery_images:
  - https://cdn.imweb.me/...
---

## 소개

소개 문단... **볼드**와 [링크](URL)도 지원됩니다.

## ODC 멤버가 되면

- **HFK 전체 커뮤니티의 일원으로 활동**: 설명
- **이벤트와 클럽에 자유롭게 연결**: 설명
- **1년 단위의 긴 호흡으로 성장**: 설명

## 포함 내용

- 2026년 **4개 시즌 이벤트** 참여 (봄/여름/가을/겨울)
- 2026년 **4개 시즌 클럽** 참여
- 시즌별 **어드벤처** 1회

## 참여 예시

봄 이벤트&클럽 → 여름 이벤트&클럽 → 가을 이벤트&클럽 → 겨울 이벤트&클럽

## 이런 분에게 추천

- 추천 대상 1
- 추천 대상 2

## ODC 멤버들의 이야기

- "멤버 후기 1"
- "멤버 후기 2"
```

**ODC 요약 설명 필드** (아임웹 '요약 설명' 칸에 표시):
| 필드 | 설명 | 예시 |
|------|------|------|
| `simple_includes` | 포함 내용 한 줄 요약 | 4개 시즌 이벤트 & 클럽 + 2개 시즌 팀 + 어드벤처 |
| `simple_comparison` | vs 4시즌 팀 비교 | 팀 2개 vs 4개 \| 어드벤처 4회 vs 없음 \| 900,000원 vs 156만원 |
| `simple_addon` | 팀 추가 안내 | 희망 시 팀 추가 등록 가능 (27만원 할인) |

**ODC 상품 종류**:
| 상품명 | 가격 | team_count |
|--------|------|------------|
| ODC 관계 | 480,000원 | 0개 시즌 |
| ODC 지성 | 660,000원 | 1개 시즌 |
| ODC 성장 | 900,000원 | 2개 시즌 |

## 상태 관리

| status | product_no | 동작 |
|--------|------------|------|
| `draft` | 없음 | **신규 상품 생성** (이미지 없으면 기본 이미지 자동 추가) |
| `draft` | 있음 | 기존 상품 **업데이트** |
| `synced` | 있음 | 기존 상품 **업데이트** |

- 동기화 성공 시 자동으로 `status: synced`로 변경됨
- 신규 상품 생성 성공 시 `product_no`가 자동으로 저장됨
- `/update-schedule`로 일정을 업데이트하면 자동으로 `status: draft`로 변경됨

---

## 주의사항

- **신규 상품 생성 시 이미지 자동 추가**: `images` 배열이 비어있으면 기본 이미지 4개를 자동으로 추가
- **category 필수**: 빈 문자열이면 동기화 스킵됨
- 동기화 전 Obsidian에서 파일 저장 필수
- HTML 생성 시 기본 이미지가 4개 미만이면 기본 이미지로 채워짐
- **링크 지원**: 소개, 세션 설명, 파트너 소개/기획 이유에서 `[텍스트](URL)` 형식의 마크다운 링크가 하이퍼링크로 변환됨
- **활동 박스 간격**: 세션 설명과 활동(회색 박스) 사이에 16px 간격이 자동 적용됨
- **활동 타입-내용 간격**: 활동 타입(워크숍, 토론 등)과 내용 사이에 8px 간격이 자동 적용됨
