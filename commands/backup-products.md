# HFK 상품 백업

아임웹의 모든 상품 정보를 Obsidian 노트로 백업합니다.

## 사용법

```
/backup-products
```

## 실행

1. 사용자에게 백업할 카테고리(시즌)를 선택하도록 질문합니다:
   - 현재 시즌: 26봄주중, 26봄주말, 추가등록, 이벤트, 시즌오프
   - 지난 시즌: 25겨울주중, 25겨울주말, 25가을주말, 25가을주중, 25여름주말, 25여름주중, 25봄주말, 25봄주중, 24겨울주말, 24겨울주중, 24가을주말, 24가을주중
   - 또는 "전체"를 선택할 수 있습니다

2. MCP Imweb 도구로 백업을 실행합니다:
   - `imweb_get_categories`로 카테고리 코드 확인
   - `imweb_get_products`로 선택한 카테고리의 모든 상품 조회
   - 각 상품에 대해 `imweb_get_product`로 상세 정보 가져오기

3. HTML 파싱하여 구조화된 데이터 추출:
   - 헤드라인, 인트로
   - 팀 배경, 운영 방향, 성장 포인트, 팀 멤버들
   - 파트너 소개
   - 3개월 세션 주제 (1~6회차)
   - 진행방식
   - 이미지 URL

4. Markdown 파일로 변환하여 저장합니다.

5. 실행 결과를 사용자에게 알려줍니다.

### 저장 위치
- `product/{카테고리}/{팀명}.md`
- 예: `product/26봄주중/영화로운일.md`

### Frontmatter 구조
```yaml
---
product_no: '상품번호'
name: '팀명'
category: '카테고리'
schedule: '화요일 19:30-22:00 (3개월)'
price: 690000
prod_status: 'sale'
status: synced
last_synced: '2025-01-29 10:00:00'
images:
  - https://cdn.imweb.me/...
---
```

## 병합 규칙 (중요)

기존 파일이 있으면 **전체 덮어쓰기하지 않고, 아래 병합 규칙을 따릅니다**:

### Frontmatter 병합
- Imweb에서 가져온 필드(`product_no`, `price`, `prod_status`, `images`)는 **Imweb 값으로 업데이트**
- `status`: `synced`로 설정
- `last_synced`: 현재 시각으로 설정
- **로컬에만 존재하는 커스텀 필드는 그대로 보존** (예: `schedule`, 사용자가 추가한 필드)

### 본문 병합
- **기존 파일에 이미 본문 내용이 있으면, 본문은 수정하지 않습니다**
- 본문이 비어있는 경우(신규 파일)에만 Imweb HTML에서 파싱한 내용으로 본문을 생성합니다
- 기존 본문의 섹션 순서, 내용, 커스텀 섹션을 절대 삭제하거나 요약하지 않습니다
- 본문을 강제로 덮어써야 하는 경우, 반드시 사용자에게 확인을 받습니다

### 금지 사항
- 기존 본문 내용을 요약하거나 축약하지 않습니다
- Imweb의 7개 표준 섹션 외에 사용자가 추가한 섹션을 삭제하지 않습니다
- 내용이 동일하더라도 포맷을 변경하지 않습니다 (줄바꿈, 들여쓰기, 마크다운 문법 등)

---

## 주의사항

- API 호출 간 0.3초, 카테고리 간 1초 딜레이가 있어 전체 백업 시 시간이 걸릴 수 있습니다
- 백업 전 dry-run 모드로 먼저 확인하려면 `--execute` 옵션 없이 실행합니다

## 결과 확인

백업 완료 후 Obsidian에서 확인:
1. `/product/{카테고리}/` 폴더 열기
2. 각 팀 노트의 frontmatter에서 `last_synced` 날짜 확인
