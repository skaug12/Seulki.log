# HFK 이미지 분류

이미지를 팀 테마에 맞게 자동 분류합니다.

## 사용법

```
/categorize-images-imweb
```

## 실행

1. 사용자에게 시즌(카테고리)을 선택하도록 질문합니다:
   - 26봄주중, 26봄주말, 25겨울주중, 25겨울주말 등

2. 해당 시즌의 팀 목록과 각 팀의 테마를 보여줍니다.
   - `product/{시즌}/` 폴더의 팀 노트에서 테마 정보 추출

3. **팀 다중 선택**: 사용자에게 이미지를 분류할 팀을 선택하도록 질문합니다.
   - 여러 팀을 동시에 선택 가능
   - "전체 선택" 옵션 제공

4. source 폴더의 이미지 목록을 확인합니다:
   - `$HFK_IMAGES/source/`

5. **이미지 리사이즈 (API 제한 대응)**:
   - source 폴더의 모든 이미지를 가로 1920px 이하로 리사이즈
   - `sips` 명령어 사용: `sips --resampleWidth 1920 {이미지파일}`
   - 가로가 1920px 이하인 이미지는 건너뜀
   - 리사이즈는 원본 파일을 직접 수정함

6. 진행 여부를 확인합니다.

7. 확인 후 **선택된 팀에 대해서만** 이미지를 분석하여 분류합니다:
   - Claude의 비전 기능으로 이미지 분석
   - 팀 테마와 가장 적합한 이미지 4개씩 선정
   - 선정된 이미지를 팀 폴더로 복사

## 폴더 구조

```
$HFK_IMAGES/
├── source/          # 분류할 원본 이미지
├── 26봄주중/
│   ├── 영화로운일/   # 팀별 폴더
│   ├── 넘버앤센스/
│   └── ...
└── 26봄주말/
    ├── AI핸즈온/
    └── ...
```

---

## 분류 프로세스

### 0단계: 이미지 리사이즈
API 요청 시 이미지 크기 제한(2000px) 오류 방지를 위해 모든 이미지를 가로 1920px 이하로 리사이즈:
```bash
# 각 이미지에 대해 가로 크기 확인 후 1920px 초과 시 리사이즈
for img in $HFK_IMAGES/source/*.{jpg,jpeg,png,webp}; do
  width=$(sips -g pixelWidth "$img" | tail -1 | awk '{print $2}')
  if [ "$width" -gt 1920 ]; then
    sips --resampleWidth 1920 "$img"
  fi
done
```

### 1단계: 팀 테마 수집
`product/{시즌}/` 폴더의 팀 노트에서 테마 키워드 추출:

| 팀 | 테마 키워드 |
|----|------------|
| 영화로운일 | 영화, 경영, 리더십, 조직 |
| 넘버앤센스 | 숫자, 데이터, 분석, 재무 |
| AI핸즈온 | AI, 기술, 디지털, 혁신 |

### 2단계: 이미지 분석
각 이미지의 내용, 분위기, 색감 분석:
- 사람이 있는지
- 비즈니스/전문적 분위기인지
- 주요 오브젝트/활동
- 색상 톤

### 3단계: 매칭 및 선정
- **선택된 팀**에 대해서만 테마와 가장 적합한 이미지 4개 선정
- 동일 이미지가 여러 팀에 중복 선정되지 않도록 처리

### 4단계: 파일 복사
- 선정된 이미지를 팀 폴더로 복사
- 원본은 source 폴더에 그대로 유지

## 주의사항

- 팀당 4개 이미지가 선정됩니다
- **팀 다중 선택**: 원하는 팀만 선택하여 분류할 수 있습니다
- **이미지 리사이즈**: 가로 1920px 초과 이미지는 자동으로 리사이즈됩니다 (원본 파일 수정)
- 원본을 보존하려면 source 폴더에 넣기 전에 별도로 백업하세요
- 이미지 분석에 Claude 비전 기능을 사용하므로 시간이 걸릴 수 있습니다
- 지원 이미지 형식: JPG, PNG, WEBP
- 이미지가 선택된 팀 수 × 4보다 적으면 일부 팀에 이미지가 부족할 수 있습니다

## 결과 확인

분류 완료 후:
1. `/images/{시즌}/{팀명}/` 폴더에서 선정된 이미지 확인
2. 필요시 수동으로 이미지 교체/추가
3. 상품 노트의 `images` 필드 업데이트 후 `/sync-products` 실행
