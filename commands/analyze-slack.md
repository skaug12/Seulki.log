# Slack 워크스페이스 분석 → Obsidian MD 저장

Slack 워크스페이스의 메시지를 수집하고 분석하여 hfk-imweb Obsidian vault의 `note/분석/` 폴더에 MD 파일로 저장합니다.

## 사용법

```
/analyze-slack [분석 주제 또는 채널명]
```

- 인자 없이 실행하면 전체 워크스페이스 종합 분석
- 채널명을 지정하면 해당 채널만 심층 분석
- 주제를 지정하면 전체 메시지에서 해당 주제 관련 내용만 추출

## 실행

$ARGUMENTS를 참고하여 분석 범위를 결정합니다.

### 1단계: 데이터 수집

Slack API를 사용하여 메시지를 수집합니다.

- 토큰: HBR Forum 워크스페이스의 Bot Token 사용
- 채널 목록 조회 → 각 채널 join → 메시지 히스토리 수집
- 사용자 정보 수집 (이름 매핑)
- Rate limit 준수 (요청 간 0.5초 대기)

### 2단계: 분석

수집된 메시지를 다음 관점에서 분석합니다:

1. **구조 분석**: 채널별 메시지 수, 활동 사용자, 시간 범위
2. **시계열 분석**: 연도별/월별 활동량 추이, 피크 시기와 원인
3. **키워드 분석**: 주요 키워드 빈도, 시기별 키워드 변화
4. **미실현 아이디어**: 논의되었으나 실행되지 않은 계획들
5. **콘텐츠 기회**: 축적된 데이터로 만들 수 있는 콘텐츠
6. **운영 갭**: 놓치고 있는 것, 못하고 있는 것

### 3단계: MD 파일 생성

분석 결과를 아래 경로에 저장합니다:

```
~/Library/Mobile Documents/iCloud~md~obsidian/Documents/hfk-imweb/note/분석/
```

파일 형식:

```markdown
---
title: '[분석 제목]'
date: 'YYYY-MM-DD'
type: output-content
source: 'Slack API 메시지 분석'
tags: [slack, 분석, ...]
---

# [분석 제목]

> **분석 기간**: ...
> **총 메시지**: ...

---

[분석 내용]
```

### 4단계: 결과 요약

- 저장된 파일 경로를 사용자에게 알림
- 핵심 발견사항 3-5개를 간략히 요약
