Introduction

Claude Code = 내 말을 이해하는 개발자

코드를 짜주는 생성형 AI

클로드 코드는 '코드'를 짜주는 genAI입니다. 기존에는 개발자가 더 효율적인 코드를 만드는 데에만 사용했어요.

하지만 우리는 다르게 씁니다. 내 컴퓨터의 파일을 관리하고, 산출물을 만드는 데 사용합니다.

"나는 기획자지만, 내 말을 모두 이해하고
문제를 해결해주는 개발자가 생긴 것."

컴퓨터 언어를 쓰지 않고, 기획자가 개발자에게 요청하듯 자연어로 말하면 됩니다.

Why Claude Code

ChatGPT, Gemini는 더 안 써도 되나요?

ChatGPT / Gemini — "알려주는" AI

  • 파일 올리면 요약해서 텍스트로 알려줌
  • 데이터가 많으면 일부만 보고 "대략" 분석
  • 결과를 복사해서 다시 붙여넣어야 함
  • "답변"을 주지만 실행까지는 안 해줌

Claude Code — "실행하는" AI

  • 내 컴퓨터의 파일을 직접 열어서 분석
  • 200개 전부 빠짐없이 처리
  • 분석 + 산출물 제작 + 반영까지 완료
  • "답변" 대신 "결과물"을 만들어 줌

Claude가 이미 포함

클로드 코드를 가입하면 Claude 대화 기능이 이미 포함되어 있습니다.

대화 기록 이전 가능

기존 ChatGPT/Gemini의 대화 기록도 수동으로 옮겨올 수 있습니다.

Perspective

반자동화 vs 완전 자동화

에이전트(Agent)와 에이전틱 AI(Agentic AI)는 조금 다릅니다.

반자동화 — Claude Code

  • 중간중간 나의 확인을 받으며 진행
  • 프로세스를 내가 점검하고 주도
  • 결과물을 검토한 뒤 다음 단계로
  • 하나의 도구를 야무지게 깊이 있게 활용

완전 자동화 — MAKE, N8N, OpenClaw

  • 정기적 · 연쇄적으로 자동 실행
  • 내가 지시하지 않아도 알아서 동작
  • 설정이 복잡하고 유지보수에 시간 소요
  • 하나가 바뀌면 전체가 깨질 수 있음

이슬기의 관점 — 매니저로서 프로세스의 주도성을 가져간다는 측면에서, 반자동화가 좀 더 안전합니다. 클로드 코드 하나를 야무지게 잘 쓰는 것이, 괜히 자동화까지 건드려 넓고 얕게 쓰는 것보다 낫다는 생각입니다.

Security

AI에게 뭘 말해도 되나요?

클로드 코드는 내 컴퓨터에서 코드를 실행하지만, 대화 내용은 클로드 서버로 전송됩니다. 따라서 보내는 내용을 구분해야 합니다.

보내도 되는 것

  • 데이터 구조 (컬럼명, 타입, 행 수)
  • 분석 코드 (실제 데이터 미포함 시)
  • 공개 정보 기반 리서치
  • 문서 초안 (민감 정보 제외)

보내면 안 되는 것

  • 실제 고객 데이터, 매출 원본
  • 내부 전략 문서, 미공개 정보
  • 하드코딩된 비밀번호 / 토큰
  • 주민번호, 카드번호, 계좌번호

.env 파일 — API 키/토큰 관리

API 키, 토큰 등 민감 정보를 코드와 분리해서 보관하는 파일. 클로드 코드에 직접 비밀번호를 적지 않고, .env에 저장 후 읽습니다.

# .env 파일 예시
GOOGLE_API_KEY=여기에_구글_API_키
SLACK_TOKEN=여기에_슬랙_토큰

.env 세팅 순서

  1. 클로드 코드에게 ".env 파일을 만들어줘" 요청
  2. 생성된 파일을 텍스트 에디터로 열고 토큰 값 직접 입력
  3. .gitignore에 .env 추가
  4. API 토큰 쓸 때 ".env 파일을 읽어서 써" 라고 요청
Philosophy

책을 사는 것만으로 나의 지식이 되지 않듯

실행해야 내 것이 됩니다

AI부사수 팀 외에도 여러 AI 강의를 들으시면, 타인의 템플릿이나 스킬, 알고리즘 등을 얻게 되실 거예요.

하지만 책을 사는 것만으로 나의 지식이 되지 않듯, 실제로 실행해보는 것이 중요합니다.

책을 읽고 작가의 의도를 파악하는 것처럼, 코드를 실행해보고 —

  • "왜 이 코드를 만들었을까?"
  • "그럼 나는 어떻게 해볼까?"

까지 고민을 전개해보시길 추천드립니다.

하나를 깊이 있게

ChatGPT, Gemini, Copilot, Claude, Cursor, n8n, Make, Zapier... 매주 새로운 도구가 등장합니다.

"하나를 정하고, 제대로 깊이 쓰자."

실행 > 수집

  • 템플릿을 모으는 것보다 하나를 직접 실행
  • 코드를 읽는 것보다 한 번 돌려보기
  • 강의를 듣는 것보다 내 업무에 적용해보기

마케터인데 디자인까지 해야 할 때?

Claude + Figma 추천

  • Figma에서 프롬프트 입력 → Claude가 디자인 생성
  • 클로드 코드와 Figma를 연결하면 파일을 직접 읽고 수정 가능
  • 데이터 시트 ↔ Figma 연동, 사이즈 변경, 디테일 수정까지
  • 이 방법으로 만든 것이 저희 세션 핸드아웃입니다

회사 보안 때문에 데이터 분석을 못하는 경우

구조만 설명 + 로컬 실행

  • 데이터셋 구조를 클로드 코드에게 설명 (열 이름, 분석 맥락 등)
  • 분석 코드를 받아서 VS Code나 RStudio 등 로컬에서 실행
  • 분석은 가능하지만 반자동화는 어려움 (데이터셋에 직접 접근해야 하므로)
