Why Claude Code

Claude Code vs ChatGPT

같은 업무를 처리할 때, 코드를 직접 실행하는 것과 대화형 분석의 차이는 뚜렷합니다.

ChatGPT (대화형 분석)

  • 파일 업로드 → 요약/분석 결과를 텍스트로 제공
  • 데이터가 크면 샘플링하거나 잘림 발생
  • 중간 과정을 검증하기 어려움 (블랙박스)
  • 결과를 다시 수작업으로 옮겨야 함

Claude Code (코드 실행형)

  • Python을 작성하고 직접 실행 → 정확한 결과
  • 전체 데이터를 빠짐없이 처리 (샘플링 없음)
  • 코드가 남아 검증/재현 가능 (투명한 과정)
  • 결과를 파일 저장, API로 바로 반영

실전 예시 — 200개 상품 분석 → 아임웹 반영: ChatGPT는 요약만 / Claude Code는 분석 → HTML → API까지 한 번에

Perspective

기획자가 개발자의 눈을 갖게 되면

"개발자가 기획자였다면 이런 것도 했을 텐데"라는 상상이 현실이 됩니다.

데이터를 직접 다루기

  • 엑셀 대신 Python으로 정확한 분석
  • 여러 소스의 데이터를 자동으로 통합
  • 분석 → 시각화 → 리포트까지 원스톱

반복 업무를 시스템화

  • 매주 하는 작업을 스킬로 자동화
  • API를 연결해 서비스 간 데이터 동기화
  • 수작업 실수를 구조적으로 방지

아이디어를 바로 구현

  • "이런 도구가 있으면 좋겠다" → 바로 제작
  • 프로토타입을 직접 만들어 검증
  • 개발팀에 의존하지 않는 실행력

핵심 변화 — "개발자에게 요청하고 기다리는 사람"에서 "직접 만들고 검증하는 사람"으로

Core Concept

커스텀 스킬

마크다운(.md) 파일 하나로 Claude에게 복잡한 업무 절차를 가르칠 수 있습니다.

만드는 법

  1. .claude/commands/ 폴더에 .md 파일 생성
  2. 파일 안에 업무 절차를 자연어로 작성
  3. Claude Code에서 /파일명 으로 실행

잘 만드는 팁

  • 단계별로 명확하게 절차를 기술
  • 입출력 형식과 예시를 포함
  • MCP 도구명을 명시하면 정확도 향상
  • 에이전트 스킬로 여러 스킬을 연결 가능
  • GitHub에 올려 팀과 공유 가능

Security

Gen AI 보안 리스크 관리

AI를 업무에 활용할 때 반드시 알아야 할 보안 원칙입니다.

공유해도 되는 것

  • 코드, 개발 관련 질문
  • 일반적인 업무 내용, 기획 아이디어
  • 마케팅 카피, 콘텐츠 초안
  • 공개된 데이터 기반 분석 요청

피해야 하는 것

  • 비밀번호, API 키, 인증 토큰
  • 주민등록번호, 계좌번호, 카드번호
  • 회사 극비 문서, 미공개 재무 데이터
  • 고객 개인정보 (이름+연락처+주소 조합)

1. 데이터 격리

민감 정보는 .env 파일로 분리, .gitignore로 업로드 차단

2. 대화는 휘발성

대화는 나와 Claude만 볼 수 있고, 끝나면 Claude도 잊어버림

3. 로컬 실행

Claude Code는 내 컴퓨터에서 실행, 파일이 외부로 전송되지 않음

Philosophy

심플하게, 이것 하나만 제대로

AI 서비스가 넘쳐나는 시대

ChatGPT, Gemini, Copilot, Claude, Cursor, n8n, Make, Zapier... 매주 새로운 도구가 등장하고, 뭘 써야 할지 오히려 혼란스럽습니다.

n8n, Make 같은 노코드 자동화 도구는 강력하지만, 설정이 복잡하고 유지보수에 시간이 듭니다.

"하나를 정하고, 제대로 깊이 쓰자."

복잡하게 만들면

  • 여러 도구를 연결하느라 시간 소모
  • 하나가 바뀌면 전체가 깨짐
  • 만든 사람만 이해하는 시스템

심플하게 만들면

  • 마크다운 파일 하나로 업무 자동화
  • 누구나 읽고 수정할 수 있음
  • 작은 것부터 시작해서 점진적으로 확장

Workshop

나만의 첫 스킬 만들기

위 5가지 중 하나를 골라 직접 만들어 봅니다.

Step 1 — 업무 선택 (5분)

  • 5가지 추천 스킬 중 하나를 선택
  • 또는 본인만의 반복 업무를 선정
  • "이 업무를 매번 어떤 순서로 하는가?"

Step 2 — 절차 작성 (10분)

  • 선택한 업무의 순서를 자연어로 적기
  • .md 파일에 단계별로 정리
  • 입력/출력이 무엇인지 명시

Step 3 — 실행 & 개선 (15분)

  • Claude Code에서 /스킬명 으로 실행
  • 결과 확인 후 절차 수정
  • 2~3번 반복하면 완성도 급상승
# 예시: 바탕화면 정리 스킬

## 실행
1. ~/Desktop 폴더의 파일 목록을 확인
2. 확장자별로 폴더 분류 (이미지/문서/기타)
3. 7일 이상 된 파일은 archive/ 로 이동
4. 정리 결과를 요약하여 보여줌

워크숍이 끝나면

  • 나만의 첫 번째 Claude Code 스킬 완성
  • 이 스킬을 기반으로 다른 업무도 확장 가능
  • GitHub에 올려 팀과 공유하는 것도 추천
